BI Design Framework — Tài liệu học thuật + DAX + Power Query · Dành cho: Giảng viên · BI Developer
1. Data Modeling — Star Schema Design Power BI Best Practice
Lý do chọn Star Schema:

⭐ Query performance: Power BI VertiPaq engine tối ưu với star schema — joins đơn giản hơn snowflake, DAX filter propagation nhanh hơn.

⭐ Readability: Chỉ 1 bảng fact, nhiều dim — dễ maintain, dễ train người mới.

⭐ Scalability: Thêm dimension mới không ảnh hưởng fact table.

⭐ DAX compatibility: Many-to-one relationships hoạt động tốt nhất với star schema.

2. Power Query — Data Cleaning Steps 14 bước xử lý
Sheet: Detail cash-in
1. Remove duplicates: Key = ID đơn hàng + ID đơn phụ
2. Creator name standardize: Trim, lowercase, remove trailing spaces (miibeo_ ✓)
3. Shop name normalize: Mapping table từ Code sheet
4. Date parsing: "Thời gian tạo đơn hàng" → Date type (dd/MM/yyyy HH:mm:ss)
5. Month extraction: = Date.Month([Thời gian tạo đơn hàng])
6. Null handling: Tiền thưởng hoa hồng → Replace null với 0
7. Data type: Giá, Số lượng, Commission fields → Decimal Number
Sheet: AFF + Bank
8. AFF merge: Join Detail cash-in-AFF vào Fact_Transaction qua Order ID
9. Campaign ID normalize: AFF1, AFF2... → Text type, no leading zeros
10. Bank date parse: TCB sheet "Ngày" column → Date type
11. Refund normalization: Số lượng bị trả lại / Số lượng bị hoàn tiền → Flag refund = true if > 0
12. Payment ID dedup: Group by Payment ID → sum amounts
13. Currency column: Remove constant "VND" column (data-ink ratio)
14. Order Status filter: Keep only "Completed" (toàn bộ đã là Completed)
// Power Query M — Creator Name Standardization
let
  Source = Excel.Workbook(File.Contents("...")),
  RawData = Source{[Name="Detail cash-in"]}[Data],
  CleanedNames = Table.TransformColumns(RawData, {
    {"Tên người dùng của nhà sáng tạo", 
      each Text.Lower(Text.Trim(_)), type text}
  }),
  RemovedDupes = Table.Distinct(CleanedNames, 
    {"ID đơn hàng", "ID đơn phụ"}),
  AddedMonth = Table.AddColumn(RemovedDupes, "Month_Order",
    each Date.Month([Thời gian tạo đơn hàng]), Int64.Type),
  FilledNulls = Table.FillDown(Table.ReplaceValue(
    AddedMonth, null, 0, Replacer.ReplaceValue,
    {"Tiền thưởng hoa hồng thực tế"}),{})
in
  FilledNulls
3. DAX Measures — Power BI Formula Library 9 measures · Production-ready 
// ===== 1. Total GMV (North Star Metric) =====
Total GMV = SUM(Fact_Transaction[Actual commission base])

// ===== 2. Total Commission =====
Total Commission = SUM(Fact_Transaction[Tổng hoa hồng thực tế])

// ===== 3. Refund Rate =====
Refund Rate % = 
DIVIDE(
  SUM(Fact_Transaction[Số lượng bị trả lại]),
  SUM(Fact_Transaction[Số lượng]),
  0
) * 100

// ===== 4. Active Creator Count =====
Active Creators = 
DISTINCTCOUNT(Fact_Transaction[Tên người dùng của nhà sáng tạo])

// ===== 5. MoM Growth — Commission =====
Commission MoM% = 
VAR CurrentMonth = [Total Commission]
VAR PrevMonth = CALCULATE(
  [Total Commission],
  DATEADD(Dim_Date[Date], -1, MONTH)
)
RETURN DIVIDE(CurrentMonth - PrevMonth, PrevMonth, 0)

// ===== 6. Commission Share % (per Creator) =====
Commission Share % = 
DIVIDE(
  [Total Commission],
  CALCULATE([Total Commission], ALL(Dim_Creator)),
  0
)

// ===== 7. Running Total GMV =====
GMV Running Total = 
CALCULATE(
  [Total GMV],
  FILTER(
    ALL(Dim_Date),
    Dim_Date[Month_Num] <= MAX(Dim_Date[Month_Num])
  )
)

// ===== 8. Bank vs TikTok Variance =====
Recon Variance = 
[Total Bank Received] - [Total TikTok Payout]

// ===== 9. Payment Accuracy % =====
Payment Accuracy % = 
VAR MatchedMonths = COUNTROWS(
  FILTER(Fact_BankRecon, ABS([Recon Variance]) < 10)
)
RETURN DIVIDE(MatchedMonths, COUNTROWS(Fact_BankRecon), 0)
4. BI Design Principles — Áp dụng từ bài giảng 8 nguyên tắc
⭐ North Star Metric
Chọn Actual Commission Base làm NSM vì: (1) phản ánh doanh thu thực sự tạo ra giá trị, (2) không bị biến dạng bởi PIT hay chia sẻ commission, (3) creator performance và product performance đều được đo qua 1 metric duy nhất.
📐 Quy tắc 5-3-1
5 KPIs: GMV, Commission, Orders, Refund Rate, Active Creators — mỗi KPI có status màu + MoM% + context. 3 câu hỏi dashboard trả lời: Doanh thu có tăng? Creator có hiệu quả? Có rủi ro không? 1 NSM: Commission Base xuyên suốt.
👁 F-Pattern Layout
NSM banner → KPI row (trái sang phải) → Trend chart (full width) → Creator + Content (half-half) → Risk heatmap. Mắt người dùng di chuyển theo chữ F: từ trái, ngang, xuống, ngang. Thông tin quan trọng nhất luôn ở top-left.
🔗 Gestalt Principles
Proximity: KPI cards group lại thành hàng. Similarity: Cùng màu accent xuyên suốt (FE2C55 = TikTok Red). Figure-Ground: Card trắng nổi trên nền xám. Continuity: Line chart thể hiện trend liên tục không đứt đoạn.
🖊 Data-Ink Ratio
KHÔNG dùng: 3D charts, pie charts nhiều slice, gridlines đậm, decorative gradients. CÓ dùng: flat colors, muted gridlines (rgba(0,0,0,0.05)), white space, minimal borders (0.5px). Mỗi pixel phải mang thông tin.
📊 Chart Selection Logic
Line chart: Trend theo thời gian (GMV/Commission 6 tháng). Horizontal bar: Ranking có tên dài (Creators, Shops). Grouped bar: So sánh 2 metrics cùng categories (Content Type). Donut: Part-of-whole (Creator share ≤ 4 categories). Heatmap table: Risk monitoring theo thời gian.
🎯 Leading vs Lagging
Lagging (kết quả): GMV, Commission, Orders — đo những gì đã xảy ra. Leading (dự báo): Active Creators, Refund Rate, Creator Concentration — cho biết performance tương lai. Dashboard balance cả 2 loại để hỗ trợ cả báo cáo và quyết định proactive.
📖 Context is King
Mỗi KPI có: (1) giá trị hiện tại, (2) MoM% so sánh, (3) context text giải thích ý nghĩa. Ví dụ: "Refund Rate 0.15%" → không đủ → thêm "T5 bất thường: 0.42% (+282%)" → người dùng hiểu ngay vấn đề cần xử lý.
5. Executive Summary — Talking Points cho thuyết trình

📌 Câu chuyện dữ liệu (6 tháng):

1. Business health: Tổng GMV 5.64 tỷ VND với commission rate 10.74% là kết quả tốt cho mô hình affiliate. Tuy nhiên trend đang giảm mạnh từ T2 (đỉnh 1.55 tỷ/tháng) xuống T6 (chỉ còn 153 triệu/tháng) — giảm 90% — là tín hiệu nghiêm trọng.

2. Creator insight: Với chỉ 3 creators mà 2 người chiếm 99.9% doanh thu, đây không phải vấn đề nhỏ — đây là rủi ro sinh tử của doanh nghiệp. HHI index 0.497 gấp đôi ngưỡng nguy hiểm 0.25.

3. Content strategy: Livestream chứng minh hiệu quả vượt trội với commission rate 16.2% so với Video chỉ 8.3%. Chiến lược nên shift sang tăng số buổi Livestream thay vì chỉ tập trung Video.

4. Financial health: Điểm sáng — đối soát ngân hàng 100% chính xác, 7/7 tháng không có chênh lệch. Hệ thống kế toán hoạt động minh bạch và đáng tin cậy.

5. Growth opportunity: AFF model tăng trưởng 104x trong 6 tháng (94K → 9.9M commission). Đây là mô hình cần được nhân rộng sang nhiều brands hơn để giảm thiểu rủi ro tập trung vào IC model.
